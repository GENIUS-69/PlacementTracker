4. 🗂️ Database Design
🔥 FINAL DECISION: Hybrid Structure (Best Choice)

Why?

Hierarchical UI needs nesting
Analytics needs flat querying

{

5. 🌱 Data Seeding Strategy (CRITICAL)
Problem

Your roadmap is huge → manual insert impossible.

Solution
Step 1: Create JSON Files
/data/
  dsa.json
  devops.json
  system_design.json
Step 2: Structure JSON like:
{
  "name": "DSA",
  "topics": [
    {
      "title": "Arrays",
      "subtopics": [
        { "title": "Two Sum" },
        { "title": "Kadane's Algorithm" }
      ]
    }
  ]
}
Step 3: Seeder Script
Clears DB
Inserts full roadmap
Sets default status = Pending
⚠️ Rule

Every yellow box = topic
Every brown box = subtopic
→ No skipping allowed

6. 🔌 API Design
6.1 Roadmap APIs
Get Full Roadmap
GET /api/roadmaps/:type
Update Subtopic Status
PATCH /api/roadmaps/:type/:topicId/:subtopicId

Body:

{
  "status": "Completed"
}
6.2 Analytics APIs
Get Progress Stats

{
  "DSA": 45,

  "SystemDesign": 20
}
Get Weekly Activity
GET /api/analytics/weekly
6.3 Study Timer APIs
Log Session

Get Study Stats
GET /api/study/stats
6.4 Test APIs
Create Test
POST /api/tests
Get Tests
GET /api/tests

7.2 Progress Calculation
